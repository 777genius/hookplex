# v1.0.6

Дата релиза: `2026-03-30`

## Почему этот релиз важен

`v1.0.6` — это релиз, в котором shared runtime-package delivery стал полноценным публичным путём для Python и Node команд, а не частичным или оговорочным workflow.

## Что изменилось для пользователей

- `plugin-kit-ai init` теперь умеет scaffold'ить Python и Node проекты вокруг `--runtime-package`
- общий путь через `plugin-kit-ai-runtime` теперь рассматривается как поддерживаемая delivery model для одобренных Python и Node lanes
- starter templates и публичные примеры обновлены под актуальные pinned versions и более ясную runtime-package историю
- release verification и version-sync checks стали жёстче, чтобы опубликованная поверхность не расходилась с задокументированным install path

## Что это значит на практике

- если команде нужен mainstream non-Go stable path, Node/TypeScript плюс `--runtime-package` теперь является first-class рекомендацией
- если команда Python-first и проект остаётся repo-local, shared runtime package теперь предпочтительнее, чем drift через vendored helper files
- `doctor` и сообщения про runtime requirements стали сильнее, поэтому проблемы окружения легче поймать до того, как обвинять сам проект

## Рекомендуемые default choices после v1.0.6

- выбирайте Go, когда нужен самый сильный production path и минимум downstream bootstrap friction
- выбирайте Node/TypeScript с `--runtime-package`, когда нужен mainstream non-Go stable path
- выбирайте Python с `--runtime-package`, когда проект repo-local и команда осознанно Python-first

## Миграционный callout

Если раньше вы считали vendored runtime helper files основным долгосрочным путём для Python или Node, перепроверьте текущие рекомендации. Для одобренных shared-package paths именно `plugin-kit-ai-runtime` теперь является более ясным поддерживаемым путём.
