# Get Help And Contribute

Use this page when you need the public path for help, bug reports, docs feedback, or focused contributions.

## Choose In 60 Seconds

- Need to report a bug or docs gap: open a GitHub issue at [Issues](https://github.com/777genius/plugin-kit-ai/issues) or [New Issue](https://github.com/777genius/plugin-kit-ai/issues/new/choose).
- Need to propose a focused fix directly: open a pull request at [Pull Requests](https://github.com/777genius/plugin-kit-ai/pulls).
- Need to report a security problem: follow [SECURITY.md](https://github.com/777genius/plugin-kit-ai/blob/main/SECURITY.md) and do not open a public issue.
- Need to understand the public contract before filing something: read [Support Boundary](/en/reference/support-boundary) and [Target Support](/en/reference/target-support).

## What This Page Is Good For

- finding the correct place for a bug report, docs fix, or contribution
- keeping community feedback tied to the public contract
- helping new contributors avoid sending the right report to the wrong channel

## Use Issues For

- public bugs in CLI, SDK, runtime helpers, or generated docs
- unclear guides, broken examples, and missing migration guidance
- feature ideas that need discussion before implementation
- support-boundary mismatches between docs and behavior

## Use Pull Requests For

- focused fixes with a clear user-facing goal
- docs clarifications that do not need a long design discussion first
- example or reference improvements that clearly align with the public product model
- tests or validation hardening around an already accepted path

## Good First Contribution Areas

- clarify onboarding and choice pages
- improve examples, starter-selection guidance, or migration guidance
- tighten generated API overview copy when the source surface changes
- fix docs or reference wording when a user-facing path feels ambiguous

## Keep Contributions Healthy

- keep one concern per pull request
- update docs when public behavior changes
- keep generated docs in sync when the pipeline says they drifted
- avoid turning a user-facing fix into a broad unrelated refactor

## Best First Stops Before You Contribute

- Need the product model first: [Managed Project Model](/en/concepts/managed-project-model)
- Need the official repo contract: [Repository Standard](/en/reference/repository-standard)
- Need the canonical authoring flow: [Authoring Workflow](/en/reference/authoring-workflow)
- Need the team rollout path: [Team Adoption](/en/guide/team-adoption)

## If You Are Unsure Where To Start

1. Reproduce the issue or gap against the public docs.
2. Check whether it is really a docs problem, a support-boundary misunderstanding, or a code bug.
3. Open the smallest useful issue or pull request that keeps the public contract clearer than before.

## Security

For undisclosed vulnerabilities, do not open a public issue. Use the private path described in [SECURITY.md](https://github.com/777genius/plugin-kit-ai/blob/main/SECURITY.md).

## Repository Contribution Guide

For the repository-level contribution notes, see [CONTRIBUTING.md](https://github.com/777genius/plugin-kit-ai/blob/main/CONTRIBUTING.md).
