# v1.0.0

Release date: `2026-03-26`

## Read This Release If

- you want the first stable public baseline for the project
- you need to know which core surfaces first became safe to treat as normal production contract
- you are comparing a newer release against the original stable launch

## What This Release Established

- the first stable public contract for the core CLI
- the approved stable Claude and Codex event set
- the first stable Go SDK integration path
- the generated scaffold rules for the supported stable paths

## Read It Through This Lens

- New repo owner: this is the baseline note that tells you what first became stable, not the current day-to-day recommendation by itself.
- Existing repo owner: use this note to understand which promises were already stable before later guidance changes.
- SDK or platform consumer: use this note as the broader stable floor, then pair it with narrower follow-up notes for your exact surface.

## Stable In This Release

- CLI: `init`, `validate`, `capabilities`, `install`, `version`
- Claude stable event set: `Stop`, `PreToolUse`, `UserPromptSubmit`
- Codex stable event set: `Notify`
- the root Go SDK integration path

## Still Beta

- optional extras from `plugin-kit-ai init --extras`
- any future area not explicitly promoted into the stable contract

## What This Did Not Mean

- It did not mean every future target or helper path suddenly carried the same stable promise.
- It did not freeze later delivery guidance for Python or Node.
- It did establish the first trustworthy public floor for the project.

## Migration Callout

SDK registration moved from root-Claude methods to platform registrars. If you were building against pre-stable shapes, check your SDK wiring before treating `v1.0.0` as the compatibility baseline.
