# `v1.0.3` Release Notes

## Summary

- release tag: `v1.0.3`
- SDK module tag: `sdk/v1.0.3`
- candidate commit SHA: `1f50cb1`
- release date: `2026-03-29`
- release type: `stable patch`
- status: `known-bad for Go SDK module consumption`

## What Went Wrong

- This release corrected the public import path and added dual SDK tagging, but it did not move the physical module root to `sdk/`.
- As a result, `github.com/777genius/plugin-kit-ai/sdk@v1.0.3` does not resolve as a normal Go module release.
- Use `v1.0.4` or newer for Go SDK module consumption.

  - not separately re-run on the `v1.0.3` tag because this patch targeted the Go SDK path rather than install-channel behavior
- polyglot-smoke:
  - not separately re-run on the `v1.0.3` tag
- generated-config/runtime-contract drift:
  - the main regression was structural: the public module path pointed at `sdk/` while the physical module root still lived at `sdk/plugin-kit-ai/`
- extended:

- Go module proxy:

  - `go get github.com/777genius/plugin-kit-ai/sdk@v1.0.3` failed for the same reason

## Known Limitations

- `v1.0.3` should not be used for Go SDK module consumption.
- The corrective structural move shipped in `v1.0.4`.
